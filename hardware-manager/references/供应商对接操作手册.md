# 供应商对接操作手册

> `~/Documents/大硬件管理/供应商管理.md` 的编辑规范、对接记录模板、分类标准与联系人采集工具。
> 入库采集标准另见 `supplier-onboarding-checklist.md`。

---

## 文档结构（两大部分，2026-05-25 重构）

| 部分 | 内容 | 编辑规则 |
|------|------|---------|
| **第一部分：供应商台账** | 按产品类型分类的供应商信息 | 新增供应商时插入对应类型章节；联系人/产品变化时更新对应字段 |
| **第二部分：对接记录** | 每次来访/拜访的交流记录 | 每次对接结束后**当天**记录，格式固定；联系人信息如有更新同步至台账 |

**编辑惯例（每次新增记录前必读）：**
- **台账优先联系人**：每次对接获取的新联系方式，立即同步更新到台账对应章节。
- **对接记录必填联系人**：即使台账联系人不全，对接记录里当天的联系人信息必须完整。
- **评估结论即时更新**：每次对接后填写评估结论，不得留"待补充"。
- **按类型不按公司**：新供应商先判断类型，插入对应节；不接受在"其他"里堆砌。

## 产品分类标准（芯谷微验证版）

- **化合物放大器**：GaAs 功率放大器、GaN 功率放大器、内匹配管放大器（IMFETs）、预匹配放大器管芯
- **滤波器**：陶瓷滤波器、SAW 滤波器等声表/陶瓷滤波器件
- **SIP/模组**：变频模块、频综模块、微波模块、T/R 组件（X/Ku/Ka 波段）、系统级封装（SIP）
- **微波控制芯片**：开关、衰减器、移相器、倍频器、混频器、VCO
- **射频芯片**：低噪声放大器、超宽带功率放大器、高线性功率放大器、多功能芯片

## 商务/资质字段（定点前必补）

技术信息之外，定点采购前供应商档案须补齐：

| 字段 | 说明 |
|------|------|
| 报价 | 关键料号单价/阶梯价 + 报价有效期 |
| 库存/货期 | 现货量、标准 lead time、MOQ |
| 账期 | 付款条件（预付/月结 N 天） |
| 资质 | 营业执照、保密资质、质量体系认证（如 ISO9001/MIL） |
| 二供 | 是否有第二来源/国产替代，对应料号 |
| 出口管制 | 是否受 EAR/实体清单影响 |

## 供应商 5 维打分卡（评估结论量化）

每次评估按 1–5 分打分，替代"待评估/初步认可"等模糊定性：

| 维度 | 1 分 | 5 分 |
|------|------|------|
| 技术匹配 | 与需求差距大 | 完全覆盖且领先 |
| 质量/资质 | 无认证 | 军工级+案例齐全 |
| 商务（价格/账期） | 明显偏高/苛刻 | 有竞争力 |
| 供货保障（货期/库存/产能） | 不稳定 | 稳定+有二供 |
| 风险（断供/管制/单点） | 高 | 低 |

> 综合 ≥4 优先定点；任一维度 ≤2 需在对接记录写明改进措施或淘汰。

## ⚠️ 发送文件前必查联系人

发送前先查 user.md 或通讯录确认收件人身份，不用已配置消息通道里的默认地址替代不同人（**发错人比发送失败更严重**）。

**macOS 通讯录本地查询（SQLite）：**
```bash
DB="$HOME/Library/Application Support/AddressBook/AddressBook-v22.abcddb"
# 查姓名
sqlite3 "$DB" "SELECT r.Z_PK, r.ZFIRSTNAME, r.ZLASTNAME FROM ZABCDRECORD r WHERE r.ZFIRSTNAME LIKE '%关键词%' OR r.ZLASTNAME LIKE '%关键词%'"
# 查电话（ZOWNER 对应 ZABCDRECORD.Z_PK）
sqlite3 "$DB" "SELECT ZFULLNUMBER FROM ZABCDPHONENUMBER WHERE ZOWNER = <Z_PK>"
# 查邮箱
sqlite3 "$DB" "SELECT ZADDRESS FROM ZABCDEMAILADDRESS WHERE ZOWNER = <Z_PK>"
```
注意：本地数据库仅存缓存数据；地址信息（ZABCDPOSTALADDRESS）通常为空（iCloud 同步不过本地）。
