# 硬件设计 8 阶段工作流 — 多 Agent 视角

胡杨 2026-06 向董事长战略汇报时确立的框架。用于"以知识库为内核的硬件 Auto-Development"主题。

## 8 个阶段

```
① 需求分析 → ② 原理图设计 → ③ PCB Layout → ④ 设计评审
                                                ↓
⑤ 仿真验证 ← ④ 结束后交叉验证
                                                ↓
⑥ 制板生产 → ⑦ 调试测试 → ⑧ 复盘与改进
```

## 每个阶段对应的 Agent 角色与知识库覆盖

| # | 阶段 | Agent 角色 | 知识库 wiki 目录 | AI 替代度 |
|---|------|-----------|-----------------|:--------:|
| ① | 需求分析 | 需求分析 Agent（系统思维+选型） | 01_硬件总体设计 + 02_器件与选型 | 60-70% |
| ② | 原理图设计 | 电路设计 Agent（模数电路+EDA） | 03_原理图设计 + 10_MCU与FPGA | <10% |
| ③ | PCB Layout | Layout Agent（SI/PI+制造） | 04_PCB设计 | <10% |
| ④ | 设计评审 | 评审 Agent（全栈知识+规范） | 跨01/03/04/09/11 | 20-30% |
| ⑤ | 仿真验证 | 仿真 Agent（电磁场+PDN+EMC） | 07_电源完整性 + 05_射频 + 09_EMC | 40-60% |
| ⑥ | 制板生产 | 制造 Agent（工艺+供应链） | 06_制造与测试 | 15-20% |
| ⑦ | 调试测试 | 调试 Agent（仪器+故障推理） | 06_制造与测试 + 08_结构与热设计 | 40-50% |
| ⑧ | 复盘与改进 | 知识管理 Agent（总结+沉淀） | 11_设计案例消化 | 80-90% |

## 知识架构五层（P2 建立的视角）

```
⑤ 制造工艺  ── SMT / 焊接 / DFM / IPC标准 / 质量管控
④ 系统集成  ── PCB层叠 / 信号完整性 / 电源完整性 / EMC / 热管理 / 结构设计
③ 功能电路  ── Buck / Boost / LDO / ADC前端 / 差分对 / 滤波器 / PLL / 射频链路...
② 器件层   ── 24大类：MCU / FPGA / 放大器 / ADC / 电源管理 / 射频 / 连接器...
① 物理底层  ── 半导体物理 / 电磁场 / 热力学 / 材料科学
```
