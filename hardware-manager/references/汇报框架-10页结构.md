## TOP3问题确定方法

**优先级：用户自身材料 > 团队直接反馈 > 典型假设**

1. 先找用户已有的分析材料（如 `硬件能力反思_*.pptx`、内部文档）
2. 用真实案例替代假设性问题
3. 三个维度示例（来自胡杨2026-05-03）：
   - P1：硬件设计能力（只顾功能，不管环境适应性/EMC/可靠性）
   - P2：流程保障能力（需求管理随意，测试闭环缺失）
   - P3：系统级分解能力弱（系统→模块需求分解不清）

## 项目状态收集流程

**两列制**：研发状态 + 订单状态

收集时逐产品线、逐项目确认，不用一次性问完。

**研发状态序列**（严格顺序）：
> 计划 → TR3 → TR4 → TR4A → TR5 → 转产 → 生命周期迭代

**特殊标注**（不在序列内）：
- 外购：单独标注，不归入研发状态序列
- "大量（但萎缩）"：作为订单状态的注释，不是独立状态

**订单状态选项**：
少量 / 中等 / 大量 / 大量（但萎缩） / 预期少量 / 预期中等 / 预期大量 / 预期海量

## 近期行动页面工作流

**当用户确认"第X页"时**：
1. 读取该页内容呈报给用户
2. 用户口述修改 → 用patch更新
3. 页面内容确认后 → 问"接下来确认哪一页？"

**近期行动页面内容规则**：
- 用户说几点就写几点，不扩展
- 用户口述的内容直接记录，不补充
- 用户可能将原有一项拆分成两项（如"规范方案评审"拆出"强制模块级测试闭环"）→ 直接按用户最新口径记录
- 节点信息（5月底/3-6个月等）由用户提供

**示例：用户口述四点**：
```
第1点：规范方案评审（评审重点：环境适应性/EMC/可靠性；强制识别风险）
第2点：强制模块级测试闭环（流程+需要的支持）
第3点：硬件设计审核环节改革（自审AI辅助；互审关注EMC/环境适应性）
第4点：电源和电池管理模块整改（缩减供应链；供电电路模块化）
```

## PPT内容读取方法

当用户提到参考某PPT文件时，用以下方法提取内容：

```python
import zipfile, xml.etree.ElementTree as ET
path = "xxx.pptx"
with zipfile.ZipFile(path) as z:
    slides = sorted([f for f in z.namelist() if f.startswith("ppt/slides/slide") and f.endswith(".xml")])
    for slide in slides:
        content = z.read(slide).decode("utf-8")
        root = ET.fromstring(content)
        texts = [t.text.strip() for t in root.iter("{...}t") if t.text and t.text.strip()]
        print(" ".join(texts))
```

注意：PPTX是单页还是多页不影响方法，直接读取所有slide XML即可。

## 内容更新原则

- 行动项必须关联对应的问题编号（P1/P2/P3）
- 措施要具体（谁、什么节点），不要空洞
- 数据占位符用 `[待统计]` / `[待确认]` 标注，不用 `[待补充]` 笼统带过
- 不用三级部门细分（三行表格即可）
